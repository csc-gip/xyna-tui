086debd6b848:~/xyna_001/server$ x help changeruntimecontextdependencies 
usage: changeruntimecontextdependencies -changes <args> [-f] [-ownerApplicationName <arg>]
       [-ownerVersionName <arg>] [-ownerWorkspaceName <arg>]
Changes runtime context dependencies.
Changes the set of runtime context dependencies in a single atomic operation.
Format for changes {ACTION}:{RUNTIME_CONTEXT_STRING_REPRESENTATION}
{ACTION}:  a or r for either addition or removal
{RUNTIME_CONTEXT_STRING_REPRESENTATION}: either the name of the workspace,
{APPLICATION_NAME}/{APPLICATION_VERSION} or {APPLICATION_DEFINITION_NAME}//{PARENT_WORKSPACE}
Examples:
... -changes a:devel
add a runtime context dependency to the devel workspace
... -changes r:prod/1.0
remove a runtime context dependency to the application 'prod' in version '1.0'
... -changes a:prod//devel
add a runtime context dependency to the 'prod' application definition defined in the workspace
'devel'
... -changes r:prod/1.0 a:prod/2.0
replace the dependency to application 'prod' in version '1.0' with a dependency on the newer version
'2.0'
   -changes <args>                      Specification of the changes to the owner runtime context.
   -f,--force                           Ignore nonfatal validation errors
   -ownerApplicationName <arg>          The name of the owner application.
   -ownerVersionName <arg>              The name of the owner application version.
   -ownerWorkspaceName <arg>            The name of the owner workspace.
